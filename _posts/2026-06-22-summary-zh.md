## [Python 自由线程：PyCon US 2026 上的过去、现在和未来](https://lwn.net/Articles/1078367/) ⭐️ 9.0/10

长期担任 CPython 核心开发者兼指导委员会成员的 Thomas Wouters 在 PyCon US 2026 上发表了全面演讲，介绍了去除全局解释器锁（GIL）的自由线程 Python 解释器的历史、现状和未来。 此次演讲标志着 Python 并发编程的范式转变，首次实现了线程的真正并行执行，可能大幅提升多核工作负载的性能。这代表了多年的社区努力，是 Python 生态系统中的一个里程碑式变化。 自 Python 3.13 起，自由线程构建已作为实验性功能提供，纯 Python 代码预计将达到与启用 GIL 时相同的线程安全程度。演讲可能涵盖了当前的采用状况、性能影响以及未来 Python 版本中使自由线程成为默认设置的路线图。

rss · LWN.net · 6月22日 15:26

**背景**: 全局解释器锁（GIL）是一种互斥锁，可防止多个线程同时执行 Python 字节码，限制了 CPU 密集型任务的并发性。几十年来，GIL 一直是 CPython 的核心特性，确保了线程安全但阻碍了真正的并行。自由线程 Python 项目旨在移除该锁，允许多个线程在多 CPU 核心上并行运行，从而显著提升某些应用的性能。
